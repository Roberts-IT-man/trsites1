About

How I
actually
work.

I got into web development because most websites genuinely frustrated me. Slow, over-engineered, built on whatever tool was trendy — and nobody could work out why visitors weren't sticking around.

So I write real code. HTML, CSS, JavaScript — whatever the job calls for. No drag-and-drop shortcuts. Every site I hand over is something I'd put my name on.

The process is kept deliberately simple. You tell me what you're after, I ask a few questions to make sure I've got the full picture, then I get on with building it. No drawn-out discovery phases before a single line gets written.


01
